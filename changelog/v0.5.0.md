## Breaking Changes
- network: Change `network.OnData` callback parameter from `network.Conn` to `interface{}` to support both `Conn` and `StreamConn` (#467)

## Features
- network: Add stream-based interfaces (`network.StreamConn`, `network.Streamer`, `network.Stream`, `network.ReceiveStream`, `network.SendStream`) for QUIC/HTTP3 support (#467)
- protocol: Add `protocol.StreamServer` interface for stream-based protocol servers (#467)
- suite: Add `suite.StreamServerFactory`, `suite.HTTP3`, `suite.HTTP3Draft29` constants (#467)
- route: Add `Engine.ServeStream`, `Engine.SetAltHeader` for stream protocol routing (#467)
- server: Add `server.WithAltTransport` option for parallel transport monitoring (#467)
- adaptor: Add `adaptor.CopyToHertzRequest` for converting `http.Request` to hertz request (#467)
- network: Add `network.DialFunc` type and `client.WithDialFunc` option (#447)
- network: Add `network.ErrorNormalization` interface for cross-platform error mapping (#412)
- protocol: Add `ResponseHeader.ContentEncoding`, `ResponseHeader.SetContentEncoding`, `ResponseHeader.SetContentEncodingBytes` (#456)
- client: Add `client.WithConnStateObserve` for host client connection pool observability (#380)
- client: Add `config.ConnPoolState` with `PoolConnNum`, `TotalConnNum`, `WaitConnNum`, `Addr` fields (#380, #493)
- errors: Add `errors.ErrNoFreeConns`, `errors.ErrConnectionClosed` sentinel errors (#380, #412)
- server: Add `server.WithBasePath` option for engine base path (#303)
- server: Add `server.WithOnAccept`, `server.WithOnConnect` options for connection lifecycle hooks (#454)
- route: Add `Engine.SetClientIPFunc`, `Engine.SetFormValueFunc` for per-engine customization (#473)

## Fixes
- http1: Fix client cannot parse stream body when transfer-encoding is 'identity' (#515)
- server: Fix `config.Options.OnAccept` panic when called before connection is wrapped (#514)

## Improvements
- http1: Try to parse response even when write errors occur (#412)
- client: Add `config.RequestOption` support to global `client.Get`, `client.Post` convenience functions (#433)
- http1: Prevent host client observation goroutine resource leak (#499)
